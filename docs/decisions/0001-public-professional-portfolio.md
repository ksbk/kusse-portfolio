# ADR 0001: Maintain drkusse.com as a Public Professional Portfolio Repository

## Status

Accepted

## Context

drkusse.com is intended to support professional visibility for employment, freelance work, research collaboration, and technical credibility. The repository should therefore show not only the final website, but also the development process.

## Decision

The drkusse.com repository will be maintained as a public professional portfolio repository. It will use documentation, issues, milestones, branches, commits, screenshots, and release notes to make the development process auditable.

Reusable template or commercial product logic may remain in a separate private or controlled repository.

## Consequences

- Employers can inspect the development process.
- The public repo demonstrates engineering discipline.
- Sensitive/private product assets must not be committed.
- Documentation quality becomes part of the professional presentation.
